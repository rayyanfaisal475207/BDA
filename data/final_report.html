Executive Summary

Traditional full-text search systems fail to maintain interactive latencies when processing massive statutory corpora. Finrate addresses this bottleneck by implementing a production-grade Locality Sensitive Hashing (LSH) and SimHash retrieval engine. By transforming high-dimensional policy documents into compact MinHash signatures, the system achieves near-constant $O(1)$ query complexity. Our implementation demonstrates a 36.6x speedup compared to exact search baselines, enabling real-time QA over thousands of academic policy sections with a minimal 7% loss in accuracy.

01. Introduction & Motivation

University handbooks (UG/PG) contain thousands of critical clauses regarding GPA requirements, attendance, and graduation criteria. Students frequently require immediate answers to complex policy questions, but manual navigation is prohibitive. Finrate bridges this gap using Big Data techniques to provide a scalable, approximate similarity search system that can quickly retrieve relevant policy sections.

03. Algorithm Implementation
3.1 MinHash and LSH

To ensure scalability, we implemented a vectorized MinHash approach using NumPy. Instead of iterating through hash functions for each token, we compute all hash values simultaneously using matrix multiplication.

# Vectorized MinHash implementation for high-speed indexing def compute_signature(self, tokens): token_hashes = np.array([self.hash_token(t) for t in tokens]) # Vectorized computation of hash functions hash_values = (np.outer(self.a, token_hashes) + self.b[:, np.newaxis]) % self.p signature = np.min(hash_values, axis=1) return signature
3.2 LSH Banding

Signatures are divided into $b=8$ bands with $r=16$ rows per band to tune the probability of collision. This allows us to find similar documents in constant time by hashing the bands into buckets.

class LSH: def query(self, query_sig): candidates = set() for band_idx in range(self.num_bands): band_sig = query_sig[band_idx * rows : (band_idx+1) * rows] bucket_key = hash(tuple(band_sig)) if bucket_key in self.buckets[band_idx]: candidates.update(self.buckets[band_idx][bucket_key]) return candidates
04. Creative Extension: Query Mining

To provide a competitive edge, we implemented a Query Pattern Mining engine using a simplified Apriori algorithm. This identifies common query patterns and "Hot Topics" (e.g., "GPA requirements", "Attendance policy") based on user interactions.

# Two-pass pattern mining for hot topic discovery def find_frequent_patterns(self, logs, min_support=0.1): # Pass 1: Count 1-itemsets (words) freq_items = self.get_frequent_items(logs, min_support) # Pass 2: Generate 2-itemset candidates and count patterns = self.generate_and_count_pairs(logs, freq_items) return patterns
